# 테이블오더 서비스 - User Stories

분류 방식: User Journey-Based
상세 수준: High-level Epic (8-12개)
Acceptance Criteria: 혼합 (핵심 플로우 Given-When-Then, 나머지 Checklist)

---

## 고객 Journey: 입장 → 메뉴 탐색 → 장바구니 → 주문 → 확인

### US-01: 테이블 자동 로그인
**As a** 고객,
**I want to** 테이블에 앉으면 별도 로그인 없이 바로 주문 화면을 볼 수 있기를,
**So that** 대기 시간 없이 즉시 주문을 시작할 수 있다.

**Acceptance Criteria:**

Given 관리자가 테이블 태블릿에 초기 설정(매장 ID, 테이블 번호, 비밀번호)을 완료한 상태에서,
When 태블릿 브라우저를 열면,
Then 저장된 정보로 자동 로그인되어 메뉴 화면이 표시된다.

- [ ] 초기 설정 정보가 localStorage에 저장됨
- [ ] 브라우저 새로고침 시에도 자동 로그인 유지
- [ ] 로그인 실패 시 초기 설정 화면으로 이동

---

### US-02: 메뉴 조회 및 탐색
**As a** 고객,
**I want to** 카테고리별로 메뉴를 탐색하고 상세 정보를 확인할 수 있기를,
**So that** 원하는 메뉴를 쉽게 찾아 선택할 수 있다.

**Acceptance Criteria:**

Given 메뉴 화면(기본 화면)에서,
When 카테고리를 선택하면,
Then 해당 카테고리의 메뉴 목록이 카드 형태로 표시된다.

- [ ] 메뉴 카드에 메뉴명, 가격, 설명, 이미지 표시
- [ ] 카테고리 간 빠른 이동 가능
- [ ] 터치 친화적 버튼 크기 (최소 44x44px)
- [ ] 메뉴 화면이 앱의 기본(홈) 화면

---

### US-03: 장바구니 관리
**As a** 고객,
**I want to** 주문할 메뉴를 장바구니에 담고 수량을 조절할 수 있기를,
**So that** 주문 전에 원하는 메뉴와 수량을 정확히 확인할 수 있다.

**Acceptance Criteria:**

Given 메뉴 화면에서 메뉴를 선택한 상태에서,
When 장바구니에 추가하면,
Then 장바구니에 해당 메뉴가 추가되고 총 금액이 실시간으로 계산된다.

- [ ] 메뉴 추가/삭제 가능
- [ ] 수량 증가/감소 조절 가능
- [ ] 총 금액 실시간 계산
- [ ] 장바구니 비우기 기능
- [ ] 페이지 새로고침 시에도 장바구니 유지 (localStorage)
- [ ] 서버 전송은 주문 확정 시에만 수행

---

### US-04: 주문 생성
**As a** 고객,
**I want to** 장바구니의 메뉴를 확인하고 주문을 확정할 수 있기를,
**So that** 원하는 메뉴를 정확하게 주문할 수 있다.

**Acceptance Criteria:**

Given 장바구니에 메뉴가 담긴 상태에서,
When 주문 확정 버튼을 누르면,
Then 주문이 서버에 전송되고 주문 번호가 표시된 후 5초 뒤 메뉴 화면으로 자동 리다이렉트된다.

- [ ] 주문 내역 최종 확인 화면 제공
- [ ] 주문 성공 시 주문 번호 표시
- [ ] 주문 성공 시 장바구니 자동 비우기
- [ ] 5초 후 메뉴 화면으로 자동 리다이렉트
- [ ] 주문 실패 시 에러 메시지 표시 및 장바구니 유지
- [ ] 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록, 총 금액, 세션 ID 포함

---

### US-05: 주문 내역 조회
**As a** 고객,
**I want to** 현재 테이블에서 주문한 내역과 상태를 확인할 수 있기를,
**So that** 주문이 제대로 접수되었는지, 현재 진행 상태가 어떤지 알 수 있다.

**Acceptance Criteria:**

Given 주문 내역 화면에서,
When 주문 목록을 조회하면,
Then 현재 테이블 세션의 주문만 시간 순으로 표시된다.

- [ ] 주문별 상세: 주문 번호, 시각, 메뉴/수량, 금액, 상태
- [ ] 주문 상태: 대기중/준비중/완료
- [ ] 현재 세션 주문만 표시 (이전 세션 제외)
- [ ] 매장 이용 완료 처리된 주문 제외

---

## 관리자 Journey: 로그인 → 주문 모니터링 → 테이블 관리 → 메뉴 관리

### US-06: 매장 관리자 인증
**As a** 매장 관리자,
**I want to** 매장 관리 시스템에 안전하게 로그인할 수 있기를,
**So that** 권한이 있는 사람만 매장을 관리할 수 있다.

**Acceptance Criteria:**

Given 관리자 로그인 화면에서,
When 매장 식별자, 사용자명, 비밀번호를 입력하고 로그인하면,
Then JWT 토큰이 발급되어 16시간 동안 세션이 유지된다.

- [ ] JWT 토큰 기반 인증
- [ ] 16시간 세션 유지
- [ ] 브라우저 새로고침 시 세션 유지
- [ ] 16시간 후 자동 로그아웃
- [ ] 비밀번호 bcrypt 해싱
- [ ] 로그인 시도 제한

---

### US-07: 실시간 주문 모니터링
**As a** 매장 관리자,
**I want to** 들어오는 주문을 실시간으로 확인하고 상태를 변경할 수 있기를,
**So that** 주문을 빠르게 처리하고 매장 운영 현황을 파악할 수 있다.

**Acceptance Criteria:**

Given 관리자 대시보드에서,
When 고객이 새 주문을 생성하면,
Then 2초 이내에 해당 테이블 카드에 신규 주문이 시각적으로 강조되어 표시된다.

- [ ] SSE 기반 실시간 주문 업데이트
- [ ] 그리드 레이아웃: 테이블별 카드 형태
- [ ] 각 카드에 총 주문액, 최신 주문 미리보기
- [ ] 주문 카드 클릭 시 전체 메뉴 상세 보기
- [ ] 주문 상태 변경 (대기중 → 준비중 → 완료)
- [ ] 신규 주문 시각적 강조 (색상/애니메이션)
- [ ] 테이블별 필터링

---

### US-08: 테이블 관리
**As a** 매장 관리자,
**I want to** 테이블 설정, 주문 수정, 고객 이용 완료 처리를 할 수 있기를,
**So that** 테이블별 주문 상태를 정확하게 관리하고 다음 고객을 맞을 준비를 할 수 있다.

**Acceptance Criteria:**

Given 관리자 대시보드에서 특정 테이블을 선택한 상태에서,
When 이용 완료 버튼을 누르면,
Then 확인 팝업 후 해당 세션의 주문이 과거 이력으로 이동되고 테이블이 리셋된다.

- [ ] 테이블 태블릿 초기 설정 (번호, 비밀번호, 세션 생성)
- [ ] 특정 주문 삭제 (확인 팝업 → 삭제 → 총액 재계산)
- [ ] 테이블 세션 종료 (이용 완료): 주문 이력 이동, 목록/총액 리셋
- [ ] 과거 주문 내역 조회 (시간 역순, 날짜 필터링)
- [ ] 각 작업에 성공/실패 피드백

---

### US-09: 메뉴 관리
**As a** 매장 관리자,
**I want to** 메뉴를 등록, 수정, 삭제하고 이미지를 업로드할 수 있기를,
**So that** 매장 메뉴를 최신 상태로 유지하고 고객에게 정확한 정보를 제공할 수 있다.

**Acceptance Criteria:**

Given 메뉴 관리 화면에서,
When 새 메뉴를 등록하면,
Then 메뉴명, 가격, 설명, 카테고리, 이미지가 저장되어 고객 화면에 즉시 반영된다.

- [ ] 메뉴 CRUD (등록/조회/수정/삭제)
- [ ] 카테고리별 메뉴 조회
- [ ] 이미지 파일 업로드 (서버 저장 후 URL 자동 생성)
- [ ] 메뉴 노출 순서 조정
- [ ] 필수 필드 검증 (메뉴명, 가격)
- [ ] 가격 범위 검증
