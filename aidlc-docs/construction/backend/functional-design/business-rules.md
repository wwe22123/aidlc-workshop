# Business Rules - Backend

---

## 1. 데이터 검증 규칙

### 인증
| 규칙 | 조건 | 에러 메시지 |
|------|------|-----------|
| BR-AUTH-01 | store_identifier 필수, 1-50자 | "매장 식별자를 입력해주세요" |
| BR-AUTH-02 | username 필수, 1-50자 | "사용자명을 입력해주세요" |
| BR-AUTH-03 | password 필수, 4자 이상 | "비밀번호를 입력해주세요" |
| BR-AUTH-04 | 로그인 5회 실패 시 15분 잠금 | "계정이 잠겨 있습니다" |
| BR-AUTH-05 | JWT 토큰 만료 16시간 | "세션이 만료되었습니다" |

### 메뉴
| 규칙 | 조건 | 에러 메시지 |
|------|------|-----------|
| BR-MENU-01 | name 필수, 1-100자 | "메뉴명을 입력해주세요" |
| BR-MENU-02 | price 필수, 0 < price <= 1,000,000 | "유효한 가격을 입력해주세요" |
| BR-MENU-03 | category_id 필수, 존재하는 카테고리 | "카테고리를 선택해주세요" |
| BR-MENU-04 | image 파일: jpg/jpeg/png/gif/webp만 허용 | "지원하지 않는 이미지 형식입니다" |
| BR-MENU-05 | image 파일: 최대 5MB | "이미지 크기는 5MB 이하여야 합니다" |

### 주문
| 규칙 | 조건 | 에러 메시지 |
|------|------|-----------|
| BR-ORD-01 | items 최소 1개 이상 | "주문할 메뉴를 선택해주세요" |
| BR-ORD-02 | quantity >= 1 | "수량은 1 이상이어야 합니다" |
| BR-ORD-03 | menu_id가 존재하고 is_available=TRUE | "현재 주문할 수 없는 메뉴입니다" |
| BR-ORD-04 | 상태 전이: PENDING→PREPARING→COMPLETED만 허용 | "허용되지 않는 상태 변경입니다" |

### 테이블
| 규칙 | 조건 | 에러 메시지 |
|------|------|-----------|
| BR-TBL-01 | table_number 필수, >= 1 | "테이블 번호를 입력해주세요" |
| BR-TBL-02 | password 필수, 4자 이상 | "비밀번호를 입력해주세요" |
| BR-TBL-03 | 매장 내 table_number 중복 불가 | "이미 존재하는 테이블 번호입니다" |
| BR-TBL-04 | 세션 종료 시 활성 세션 존재 필수 | "활성 세션이 없습니다" |

## 2. 비즈니스 제약조건

| ID | 제약조건 | 설명 |
|----|---------|------|
| BC-01 | 단일 매장 운영 | seed data로 1개 매장만 생성 |
| BC-02 | 단일 관리자 | seed data로 1개 관리자 계정만 생성 |
| BC-03 | 테이블당 활성 세션 최대 1개 | 동시에 2개 이상 활성 세션 불가 |
| BC-04 | 주문 번호 유일성 | 시스템 전체에서 주문 번호 중복 불가 |
| BC-05 | OrderItem 스냅샷 | 주문 시점의 메뉴명/단가를 저장 (메뉴 변경 영향 없음) |
| BC-06 | 세션 종료 시 이력 보존 | 주문 데이터를 OrderHistory로 이동 후 원본 삭제 |
| BC-07 | 메뉴 soft delete | 활성 주문에서 참조 중인 메뉴는 is_available=FALSE 처리 |

## 3. 에러 처리 규칙

| 상황 | HTTP Status | 처리 |
|------|------------|------|
| 리소스 없음 | 404 | "해당 리소스를 찾을 수 없습니다" |
| 인증 실패 | 401 | "인증에 실패했습니다" |
| 권한 없음 | 403 | "접근 권한이 없습니다" |
| 입력 검증 실패 | 422 | 필드별 에러 메시지 반환 |
| 비즈니스 규칙 위반 | 400 | 규칙별 에러 메시지 반환 |
| 서버 내부 오류 | 500 | "서버 오류가 발생했습니다" |

### 에러 응답 형식
```json
{
  "detail": "에러 메시지",
  "error_code": "BR-XXX-XX"
}
```
